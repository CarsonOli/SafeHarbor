# SafeHarbor Admin Guide (Non-Technical)

## Who this guide is for

This guide is for program managers, coordinators, and operations staff who use the admin portal but do not manage software infrastructure.

---

## 1) User management

### Add a user

1. Open **Admin > User Management**.
2. Select **Add user**.
3. Enter work email and assign one role:
   - **Admin**: full system administration.
   - **SocialWorker**: case and resident workflows.
   - **Viewer**: read-only reporting access.
4. Save and confirm the person can sign in.

### Update a user

1. Find the user by email.
2. Choose **Edit**.
3. Update role/status.
4. Save changes.

### Disable a user (recommended over deletion)

1. Open user profile.
2. Set status to **Inactive**.
3. Save.

> Why: Disabling preserves history and audit records.

---

## 2) Standard CRUD workflows

### Create records

- Use **New** buttons inside each module.
- Complete all required fields (marked with `*`).
- Confirm details before saving.

### Read/search records

- Use filters first (status/date/owner).
- Avoid broad exports unless requested by leadership.

### Update records

- Change only fields you are authorized to edit.
- Add notes/context when prompted.

### Delete records

- Use deletion only when policy allows (see safe deletion section below).

---

## 3) Safe deletion rules

Deletion is high-risk and should be rare.

1. Prefer **Archive** or **Inactive** when available.
2. Do **not** delete records tied to:
   - legal holds,
   - active case investigations,
   - donor financial reconciliation periods.
3. Use the two-person check:
   - Requestor documents reason.
   - Admin approver confirms policy eligibility.
4. Record deletion reason in the provided notes field.

If unsure, stop and escalate to the operations director.

---

## 4) What to do when something looks wrong

1. Capture screenshot and record exact time.
2. Note page/module and what action failed.
3. Contact support channel with:
   - who was affected,
   - urgency,
   - whether work is blocked.

Do not retry destructive actions repeatedly.
